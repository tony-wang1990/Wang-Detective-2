# Wang-Detective 项目状态与 UI 约定

更新时间：2026-05-26

本文档用于记录当前改造进度、后续计划和 UI 集成方式。后续新增功能优先按本文档约定实现，避免再出现“功能已经做了但入口藏在独立页面里”的体验问题。

## UI 集成约定

| 约定项 | 当前标准 | 后续要求 |
| --- | --- | --- |
| 主框架 | 保留原 Web 面板 `/dashboard/*` 的左侧导航 + 顶部栏 + 右侧内容区结构 | 新功能优先接入这个主框架，不做割裂式外部页面 |
| 菜单入口 | 左侧菜单是主入口 | 新功能必须有清晰菜单项，名称要能直接说明用途 |
| 页面切换 | 点击菜单后在右侧内容区切换 | 不默认整页跳转，不默认打开新窗口 |
| 独立页面 | 旧版完整控制台保留为 `/legacy-dashboard.html` | 仅作为排障、直接访问或降级入口 |
| 当前实现方式 | `frontend/` 已成为可维护 Vue 源码，生产入口使用 Vue 路由 | 新功能继续按 Vue 页面/组件接入，不再新增割裂式独立页面 |
| 版本显示 | 当前版本和最新版本改为基于 `Wang-Detective/main` 提交号，例如 `main-225b96e` | 每次 Bug 修复和镜像重建都应能追踪到提交号，不再显示 `null` |
| 低配 VPS | 默认 1C/1G 友好参数，watcher 默认轻量运行 | 后续功能避免默认启动额外常驻重任务 |

## 下一阶段建议顺序

1. 真实 OCI 高危动作验收：终止、改安全规则、拆卷救援、恢复回滚必须用专用测试资源逐项确认。
2. 前端全交互打磨：按钮 loading、成功/失败反馈、空状态、错误解释、暗色模式和移动端。
3. 运维入口产品化二期：SFTP 大文件提示、主机复制/批量导入、端口转发说明。
4. Object Storage 和备份归档：把数据库、日志和配置备份形成云端闭环。
5. 权限体系：只读、运维、管理员角色和高危动作审批。

## 后续开发提醒

- 新功能不应只加接口或独立 HTML，必须同步考虑主面板入口。
- 低配 VPS 是主要部署环境，默认功能不能明显增加 CPU/内存常驻压力。
- 涉及密码、私钥、Token 的功能必须加密存储或只短时驻留内存。
- 涉及 SSH、SFTP、删除、批量命令、更新等操作必须写审计。
- 每次完成一阶段后，应推送到 `Wang-Detective`，等待 GHCR 镜像构建完成，再让服务器部署验证。
