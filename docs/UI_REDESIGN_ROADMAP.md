# Wang-Detective UI 重设计路线

更新时间：2026-05-26

本文件记录新版 UI 的既定方向、当前完成进度和后续拆分步骤。后续新增功能优先按这里的框架走，避免出现功能能用但入口、风格和主面板割裂的问题。

## 已确定的新 UI 方向

| 项目 | 约定 |
| --- | --- |
| 整体风格 | 现代运维控制台，不再沿用高饱和蓝紫渐变和装饰光效 |
| 主色 | 深石墨侧边栏、浅灰工作区、白色内容面板 |
| 强调色 | 青色用于主操作和当前状态，琥珀色用于更新/提醒，红色只用于危险动作 |
| 组件形态 | 低圆角、低阴影、高信息密度，适合长期运维扫描 |
| 页面结构 | 继续保留左侧导航、顶部状态栏、右侧内容区 |
| 新功能入口 | 必须进入主面板内容区，不能默认弹出独立页面 |
| 低配 VPS | UI 优化不引入额外常驻重任务，不增加服务端压力 |

## 下一步建议

1. 部署最新镜像后先跑 `scripts/remote-smoke-test.sh`，确认公网域名、旧入口迁移、风险看板、VCN 和安全规则接口都通过。
2. 继续用真实 OCI 测试资源做高危动作验收：终止、改安全规则、拆卷救援、恢复回滚。
3. 前端下一轮重点放在真实机型细节验收：暗色模式、按钮反馈、空状态、错误提示、长文本抗卡字和高危弹窗。
